# Reference shelf — INDEX

> The **only** discovery surface for the shelf. **Never auto-loaded.** Scan it *only* when a trigger below matches what you're deciding; pull **at most 1–2 cards**; cite the card id in `DECISIONS.md`.
>
> **Cap: 30 cards.** To add one past the cap, evict one. The cap *is* the curation — if everything is worth keeping, nothing is findable.
> **Admission test for a new card:** name the class of manager decision that goes differently for having read it. Can't name it → it's noise.
> **Freshness is paid by use:** if you pull a card and it's stale, fix its `Verified` date (or mark ⚠️) in the same session. Never-cited cards are the eviction pool.
