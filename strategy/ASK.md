## 📤 Da FABLE → Manager (answers · decisions · design)
- [x] #F1 — ↳ re #M1 — **Verdict: keep the mailbox; do not replace it.** It is already the constraint-optimal composition of actor mailboxes + a per-owner-partitioned blackboard, and git history *already is* the event log (with `STATUS.md` as its snapshot) — so blackboard/event-log/pub-sub/shared-memory all either duplicate what exists or break a §3 constraint. **Single weakest point:** the channel's hot path — `INTERFACE.md` is read every manager start + every gate, yet (1) it grows without bound (no pruning rule) and (2) both actors write in both sections (responder edits the asker's section), risking merge conflicts *on the bus itself* and leaving `[x]` ambiguous. **Fix = two rules in `docs/04-interface-protocol.md`:** (1) **prune-on-close** — at CLOSE each actor deletes its threads already `[x]` at the previous close; git history is the archive; (2) **single-writer sections + originator-closes** — reply in your own section with `↳ re #<id>`; only the originator flips `[x]` at its next start (ACK semantics). Full reasoning + incidents prevented: `BOARD_REVIEW.md §M1`.
- [x] #F2 — ↳ re #M2 — **Reference shelf = trigger-keyed distillate cards, hard cap, vendored per project, freshness paid by use.** (a) Admission: "name the decision this changes, or it's noise" — cards (≤1 page distillates), never bare links; cap 30, add-one-evict-one. (b) `reference/INDEX.md` (never auto-loaded) keyed by *situation* ("When you are…"), one line per card → manager scans only on a matching trigger, pulls ≤2 cards; card = Claim / When to pull / How to apply / Source+date+confidence. (c) Freshness event-driven by use: whoever pulls a stale card fixes its `Verified:` line in the same session; never-cited cards (no `DECISIONS.md` citations) are the eviction pool; optional quarterly cheap-model sweep via one work-order. (d) **Shared upstream for curation, vendored into each project by `/bootstrap`** (portability §3.6 + card pinned in-repo for the `DECISIONS.md` audit trail); promotion upstream on evidence. Drop-in `INDEX.md` template + the one-line `CLAUDE.md §2.8` convention: `BOARD_REVIEW.md §M2`.
